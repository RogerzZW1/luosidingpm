# 竞品调研方法

> 本文件用途：定义竞品调研的输入、证据来源、页面访问方法、功能拆解方法和最终报告格式。该方法要求调研必须可复现、可引用，不允许只输出抽象总结。

## 输入来源
1. 产品网址
2. 产品截图或录屏
3. 产品名称与功能关键词
4. 用户已有调研笔记
5. 可选：测试账号

## 调研工作流
### 阶段 1：确定范围
明确要调研的竞品、功能范围、目标问题。例如：比较客服机器人中的“转人工流程”。

### 阶段 2：收集直接证据
优先从以下位置收集：
1. 官网首页：判断产品定位、目标用户、价值主张。
2. 功能页 / 解决方案页：判断主打功能和核心机制。
3. 定价页：辅助判断目标客户层级与商业模式。
4. Help Center / FAQ / Docs：补充边界、限制条件、流程细节。
5. 产品实际页面：用浏览器技能访问、导航、截图、记录入口和主流程。

### 阶段 3：收集外部证据
固定搜索来源：
1. Google：`产品名 + 功能关键词`
2. Google：`产品名 + review`
3. Reddit：`产品名 + pain points / alternatives / review`
4. YouTube / 博客评测：找实际演示视频和深度对比文章

### 阶段 4：页面和功能拆解
每个竞品至少拆：
1. 产品定位：通过首页标题、副标题、功能页和定价页综合判断。
2. 目标用户：通过首页文案、行业案例、套餐设计、客户 logo 判断。
3. 功能入口：通过导航、页面 CTA、帮助中心截图判断用户从哪里进入该功能。
4. 主流程：通过浏览器访问、录屏或帮助中心步骤还原操作链路。
5. 状态与反馈：通过页面截图、录屏和帮助中心说明识别默认态、处理中、成功、失败、空态等。
6. 异常与边界：通过 FAQ、社区吐槽、帮助中心限制条件、错误提示截图提取。

### 阶段 5：得出结论
结论必须基于证据，不允许只给感受。至少输出：
1. 产品定位
2. 核心机制
3. 优势与劣势
4. 对我方产品的借鉴点和风险点

## 浏览器执行规范
- 优先使用 `agent-browser` 或 `playwright-interactive` 浏览器技能做页面访问、点击、导航和截图。
- 浏览器技能负责：访问页面、记录路径、截取关键页面。
- 本调研方法负责：解释网页证据、拆功能、形成对比结论。

## 输出格式
### 产品层结论（文字）
1. 产品定位
2. 核心机制
3. 优势
4. 劣势
5. 借鉴点
6. 风险点
